# Rúbricas de Evaluación - Plan de Mejoramiento 
## Desarrollo Backend API REST con NestJS

---

## 1. Rúbrica: Entrega de la API REST Funcionando
**Duración: 30 minutos**

| Criterio | Excelente (5) | Bueno (4) | Satisfactorio (3) | Necesita Mejora (2) | Insuficiente (1) |
|----------|---------------|-----------|-------------------|---------------------|------------------|
| **Repositorio y Documentación** | Repositorio completo en GitHub con README detallado, documentación de API y instrucciones claras | Repositorio completo con documentación básica pero funcional | Repositorio con archivos principales, documentación mínima | Repositorio incompleto, documentación deficiente | Sin repositorio o documentación inexistente |
| **Endpoints CRUD Completos** | Todos los endpoints CRUD funcionan perfectamente para las 4 entidades (estudiantes, cursos, profesores, asistencias) | La mayoría de endpoints funcionan correctamente | Endpoints básicos funcionan, algunas limitaciones | Pocos endpoints funcionan correctamente | Endpoints no operativos o con fallas críticas |
| **Base de Datos** | Base de datos bien estructurada, relaciones correctas, datos de prueba completos | Base de datos funcional con estructura adecuada | Base de datos básica pero operativa | Base de datos con limitaciones estructurales | Base de datos no funcional o mal estructurada |
| **Validaciones y Manejo de Errores** | Validaciones robustas, manejo completo de errores HTTP, respuestas consistentes | Validaciones básicas, manejo adecuado de errores principales | Validaciones mínimas, manejo básico de errores | Validaciones limitadas, errores mal manejados | Sin validaciones o manejo de errores |
| **Pruebas con Postman/Thunder** | Colección completa de pruebas, todos los casos cubiertos, documentación clara | Pruebas principales funcionando, documentación básica | Pruebas básicas operativas | Pruebas limitadas o incompletas | Sin pruebas o no funcionan |

**Peso de la evaluación: 30%**

## 2. Rúbrica: Desarrollo del Proyecto desde Cero en Tiempo Real
**Duración: 2.5 horas**

| Criterio | Excelente (5) | Bueno (4) | Satisfactorio (3) | Necesita Mejora (2) | Insuficiente (1) |
|----------|---------------|-----------|-------------------|---------------------|------------------|
| **Configuración del Entorno** | Configura Node.js, NestJS CLI, dependencias y estructura de proyecto expertamente | Configuración correcta con buenas prácticas | Configuración básica pero funcional | Configuración con algunos errores menores | Configuración deficiente o incorrecta |
| **Diseño de Base de Datos (Prisma)** | Diseña esquema Prisma óptimo, relaciones correctas, migraciones adecuadas | Diseño de esquema funcional con relaciones apropiadas | Diseño básico de esquema que cumple requisitos | Diseño de esquema con limitaciones menores | Diseño de esquema deficiente o incorrecto |
| **Modelos de Datos (Prisma)** | Implementa modelos Prisma complejos, relaciones avanzadas, validaciones robustas | Modelos bien estructurados con relaciones básicas | Modelos funcionales que cumplen requisitos | Modelos básicos con limitaciones | Modelos deficientes o no funcionales |
| **Controladores y Servicios (NestJS)** | Implementa controladores y servicios bien organizados, inyección de dependencias correcta | Controladores y servicios funcionales con buena estructura | Controladores y servicios básicos pero operativos | Controladores y servicios simples con limitaciones | Controladores y servicios deficientes o mal estructurados |
| **Implementación CRUD** | CRUD completo, eficiente, con manejo avanzado de casos edge | CRUD funcional para todas las entidades | CRUD básico que cumple requisitos | CRUD limitado con errores menores | CRUD deficiente o incompleto |
| **DTOs y Validaciones** | DTOs complejos, validaciones con class-validator, transformaciones optimizadas | DTOs básicos, validaciones funcionales | DTOs mínimos pero operativos | DTOs limitados | Sin DTOs o validaciones deficientes |
| **Explicación del Proceso** | Explica cada concepto claramente, demuestra comprensión profunda de arquitectura | Explica la mayoría de conceptos con buena comprensión | Explica conceptos básicos adecuadamente | Explicación limitada, comprensión básica | Explicación deficiente o incorrecta |

**Peso de la evaluación: 50%**

| Puntuación | Calificación | Descripción |
|------------|--------------|-------------|
| 4.5 - 5.0 | Excelente | Supera las expectativas, dominio completo |
| 3.5 - 4.4 | Bueno | Cumple las expectativas, conocimiento sólido |
| 2.5 - 3.4 | Satisfactorio | Cumple los requisitos mínimos |
| 1.5 - 2.4 | Necesita Mejora | Por debajo de las expectativas |
| 1.0 - 1.4 | Insuficiente | No cumple los requisitos mínimos |
